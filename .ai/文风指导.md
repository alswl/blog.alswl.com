# content/ 文章文风指导

## 一、分析结论概览

博客时间跨度约 **2008–2025**，体裁包括：**年终总结、技术长文、读书笔记、生活随笔、工具/清单文**。整体文风偏**第一人称、坦诚、有结构**，技术文偏**问题驱动与可操作**，生活/读书偏**有温度、有反思**。

---

## 二、Frontmatter 规范

**必选字段：**

- **title**：中文标题，可带副标题（如「李飞飞如何获得成功？ - 读「我看见的世界」」）。
- **slug**：英文短标识，用于 URL，近年多用小写连字符（如 `2024-review`、`architecture-design-the-easy-way`）。
- **date**：ISO 8601，带时区，如 `2025-01-11T13:52:48+08:00`。
- **categories**：单或复数，如 `["life"]`、`["coding"]`、`["Reading"]`、`["thinking"]`、`["fun"]`。
- **tags**：数组，小写或首字母大写，如 `["review"]`、`["gtd", "obsidian-tasks"]`。

**常用可选：**

- **typora-copy-images-to**：`../../static/images/YYYYMM`，与发文月份一致，便于图片管理。
- **author**：部分文章有 `author: "alswl"`，非必须。

**约定：**

- 文件名：`YYYY-MM-DD-slug.md`，与 `date` 的日期一致或接近。
- 技术/长文可加 `draft: true` 先存稿。

## 三、文章结构习惯

### 3.1 层级与节奏

- **一级标题（##）**：大板块（如「生活」「工作」「读书」「概览」「最后」）。
- **二级标题（###）**：子主题或步骤（如「好奇心是原动力」「需求看用例」「案例」「小结」）。
- 技术长文常见模式：**概览 → 概念澄清 → 分节（每节内：论述 → 案例 → 小结）→ 番外/书单 → 最后**。
- 年终总结：固定板块 **生活 / 工作 / 业余 / 读书**，再加 **Flag**、**Last** 或「往年总结」链接。

### 3.2 开头方式

| 类型      | 典型开头                                                                                  |
| --------- | ----------------------------------------------------------------------------------------- |
| 年终总结  | 时间感或主题句（如「地球不停息绕太阳一周…」）、或直接进入第一板块                         |
| 技术文    | 背景/问题/动机，或分享缘起（如「前几日团队内部技术分享…」「将这些实践凝练成 20 字口诀」） |
| 读书笔记  | 名言引用（`> 读一本好书…`）或个人轶事（如「第一次听闻李飞飞的名字，是在…」）              |
| 生活/短文 | 直接叙事（如「前天上午从家里赶到南京…」）                                                 |

### 3.3 结尾方式

- **Last** / **最后**：一句收束或新年期望（如「期望新的一年可以热烈又恬静，深刻又朴素」）。
- 技术连载：可写「下篇文章再讲…」。
- 读书笔记：可回到个人问题（如「你找到你的北极星了么？」）或金句。
- 年终总结：可附「往年总结」列表链接。

## 四、语气与用词

- **人称**：以「我」为主，少量「我们」用于团队/读者共情。
- **基调**：坦诚、略口语但不随意；技术文保持专业、可操作。
- **技术文**：问题驱动、有明确结论；善用「小结」和要点列表；观点用因果支撑（「因为…所以…」「关键在于…」）。
- **生活/年终**：有具体细节（地点、数字、小事）；有反思与转折（「痛苦总是让人深思…」「好在…」）。
- **读书笔记**：个人体会与书中观点交织；常用「就我而言」「对我而言」「难能可贵的是」。
- **避免**：空洞堆砌、过度正式、无个人视角的纯摘抄。

## 五、格式与排版

### 5.1 图片

- 语法：`![alt](path)`，path 多用 `../../static/images/YYYYMM/xxx`。
- 图注：统一用 `<small>...</small>` 紧跟图下（如 `<small>2024 新年伊始，海边的灯塔。</small>`）。
- 图片来源：可在图注中写「image via [Pixabay](url)」或「via [某博客](url)」。

### 5.2 强调与高亮

- **关键概念/结论**：`<mark>...</mark>`（如「架构看问题」「需求看用例」）。
- **术语/要点**：`**粗体**`。
- 技术文里「口诀」「核心定义」多用 mark 或粗体，便于扫读。

### 5.3 引用

- 名言/定义：用 `> ` 块引用；可多行。
- 出处：在引用后或文末用 `[书名 (豆瓣)](url)` 等形式。

### 5.4 列表

- 无序：`-`；有序：`1.`。
- 技术文：步骤、要点、工具名、书单常用列表；年终「最受欢迎内容」等用 `-` + 链接。

### 5.5 链接

- 豆瓣：`[书名 (豆瓣)](https://book.douban.com/subject/xxx)`；音乐同理。
- 外链：`[描述](url)`，必要时注明「via」「来自」。
- 站内：可用相对或绝对路径，如 `[往年总结](https://blog.alswl.com/...)`。

### 5.6 其他

- 摘要：长文可在首段后加 `<!-- more -->` 作列表摘要截断。
- 中英文混排：专有名词、产品名保留英文（如 ImageNet、GTD、Obsidian Tasks）；术语首次出现可中英并列。

## 六、按类型的写作要点

### 6.1 年终总结

- 固定板块：**生活 / 工作 / 业余 / 读书**；每块配图 + 短图注，生活可多图叙事。
- 生活：地点、陪伴、具体小事（如「周三、周五家庭日」「迪士尼，年末的花火」）。
- 工作：真实处境 + 反思（压力、角色变化、心态），可引用自己的推文/文章。
- 业余：开源项目、Side Project、社交（Twitter/X）、播客等，可列「最受欢迎内容」带链接与数据。
- 读书：每条 `[书名 (豆瓣)](url)` + 封面图 + **短评**（一两句感受或启发），不必每本长评。
- 结尾：**Flag**（去年执行情况 + 新年 Flag）+ **Last**（一句期望）+ 往年总结链接。

### 6.2 技术长文

- 开篇：问题/背景/分享缘起；可先给「口诀」或结论（如 20 字口诀）。
- 结构：概念澄清（是什么、边界）→ 分节论述；每节内可含「案例」「小结」。
- 善用：`<mark>`、列表、小标题；必要时配架构图/流程图，图注 `<small>`。
- 结尾：总结清单（✅ 问题定义、✅ 产出物…）；可带「番外」如画图工具、书单。

### 6.3 读书笔记

- 开头：名言 `>` 或与书相关的个人经历，再点出书的价值。
- 正文：按书的结构或自拟主题分节（如「如何获得成功」「作为父母如何教育小孩」）；摘录与解读结合，用 **粗体** 标书中关键概念。
- 结尾：回到个人（如「你找到你的北极星了么？」）或金句收束。
- 引用：书中原话用 `>`；文末可附豆瓣链接或参考链接。

### 6.4 生活/随笔

- 可极短（几段 + 一图），也可按时间/主题分小节。
- 保持具体：时间、地点、人物、小事；结尾可一句点题（如「The Happy Ending」「今天开始在新公司上班，新的开始」）。

### 6.5 清单/工具文

- 开篇简短说明动机与用法（如「每次都要从豆瓣复制标题去 Apple Music 搜索…于是整理了这批链接」）。
- 清单项结构统一：标题 + 元信息 + 图 + 链接（如 ▶️ Apple Music）；可带一句 Update 说明后续变更（如 Random Apple Music 网页）。

## 七、可复用的模板片段

**年终总结 - 读书一条：**

```markdown
[书名 (豆瓣)](url)
![封面](../../static/images/YYYYMM/xxx.jpg)
一两句短评，可带个人触动或与其它书/经历对比。
```

**技术文 - 小节收尾：**

```markdown
### 小结

- 要点一
- 要点二
  本阶段产出：xxx。
```

**开篇引用（读书）：**

```markdown
> 读一本好书，就是和许多高尚的人谈话 - 歌德
```

## 八、风格一致性检查清单（写后自检）

- [ ] Frontmatter：title / slug / date / categories / tags 完整；图片目录 `typora-copy-images-to` 与月份一致。
- [ ] 图片：每图下有 `<small>` 图注或来源。
- [ ] 重点句：关键结论/术语用了 **粗体** 或 `<mark>`。
- [ ] 链接：豆瓣/外链格式统一，必要处有「via」或说明。
- [ ] 结尾：有明确收束（Last/小结/金句或下篇预告）。
- [ ] 人称与语气：以「我」为主，技术文有因果与小结，生活/读书有个人感受。

以上规范来源于对 `content/posts` 下多篇代表性文章（如 2024/2023 年终总结、架构设计 the Easy Way、李飞飞读书笔记、Code Review、死海效应、GTD、豆瓣 Top250、乱码之战、少有人走的路、The Happy Ending 等）的归纳，可直接用作后续写作与修订的参考；若希望某一块（如仅技术文或仅年终总结）单独成文档，可再拆成独立小节或单独文件。
