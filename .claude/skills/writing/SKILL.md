# 博客写作规范

基于本站已有文章归纳的写作风格与格式约定，用于保持统一语气、结构和排版。

## 适用范围与原则

- **何时用**：撰写或修改 `content/posts/` 下的新文章、年终总结、技术/读书笔记时，按本规范执行。
- **原则**：中文为主、第一人称（我）、真诚有立场；技术文简洁实用，生活/读书可带适度文学性；不堆砌、不敷衍。

## Frontmatter

每篇正文前必须包含 YAML frontmatter，字段约定如下：

```yaml
---
title: "文章标题"
slug: "url-slug"
date: 2025-02-04T12:00:00+08:00
categories: ["coding"]
tags: ["tag1", "tag2"]
typora-copy-images-to: ../../static/images/202502
---
```

- **必填**：`title`、`slug`、`date`、`categories`、`tags`、`typora-copy-images-to`
- **slug**：英文短横线，与 URL 对应，如 `architecture-design-the-easy-way`
- **typora-copy-images-to**：按年月 `../../static/images/YYYYMM`，与正文图片路径一致
- **可选**：`draft: true`、`author: "alswl"`

## 正文结构

### 技术 / 方法论文

- **开篇**：直接点题或一句金句/引用，可配图；必要时在首段说明「本文仅针对…」「不涉及…」等范围与免责。
- **概述**：用「概述」「概览」节简要说明要解决的问题与方案要点（可加粗核心词）。
- **主体**：用 `## 大标题` 分节，`### 小标题` 分小节；每节先结论再展开，善用列表和表格。
- **收尾**：用「最后」或「小结」总结；可加「扩展阅读」列出豆瓣/文档链接。

### 年终总结

- 按板块分节：`## 生活 - 简短副标题`、`## 工作 - 副标题`、`## 业余`、`## 读书` 等。
- 每块可配图，图下用 `<small>图注</small>`。
- 读书可列书名+豆瓣链接+一两句感悟；可带封面图。
- 文末用「Last」或「Flag」收束，可链往年总结。

### 读书笔记 / 感悟

- 开篇可用 blockquote 金句，再进入「第一次…」「如今…」等个人视角。
- 用 `### 小标题` 提炼维度（如「好奇心是原动力」「逆境中的坚韧」），每段有明确观点。
- 结尾可回到「北极星」「启示」等升华，保持克制不空泛。

### 教程 / 短文

- 结构简单：原因 → 准备 → 过程 → 总结；或直接分步骤。
- 过程可用 `<center><mark><b>步骤要点</b></mark></center>` + 图片；列表写工具/材料。

## 语言与语气

- **人称**：以「我」为主，表达个人判断与经验（「我认为」「我推荐」「我体感」）。
- **立场**：有明确观点，可写「实用」「简单」「the Easy Way」等定位；技术文少废话，多结论。
- **术语**：中文叙述，专业术语保留英文并在首次出现时自然带出（如 REST、GitOps、MVP、OKR）。
- **口语化**：可偶用口语（如「逼兜」「拉了一泡稀」）但整体保持克制，不破坏可读性。

## 格式约定

- **强调**：重要概念用 **粗体**；需高亮的一句用 `<mark>...</mark>`。
- **图片**：
  - 路径：`![alt](../../static/images/YYYYMM/xxx.png)`，与 `typora-copy-images-to` 对应。
  - 图注：图下紧跟 `<small>图注或来源</small>`。
- **引用**：段首金句或他人观点用 `> blockquote`。
- **链接**：`[正文](url)`；外链可注明来源如 `(豆瓣)`、`(X)`。
- **列表/表格**：多选项、对比、清单用列表或表格，保持层级清晰。

## 收尾习惯

- 技术/方法论文：以「最后」段总结，可加「扩展阅读」列表（书名/链接+一句话）。
- 年终总结：以「Last」或「Flag」+ 往年总结链接。
- 教程/短文：可简单一句总结或「若有收获，就点个赞吧」类收束。

## 示例片段

**技术文开篇：**

```markdown
当开始创建一个新系统，或参与一个新团队或项目时，都会面临一个简单却深刻的问题：这个系统的 API 是否有设计规范？
这个问题困扰了我很长时间。因此，我想整理出一份**简单而实用**的 Web API 最佳实践，也就是本文。
```

**总结文板块：**

```markdown
## 工作 - 基建狂魔

去年，我从一名探索者转变为有组织推进系统演进的人。... 痛苦总是让人深思...
```

**图片与图注：**

```markdown
![arch-easy](../../static/images/202307/arch-easy.png)
<small>image via [Pixabay](https://...)</small>
```

按以上规范写出的文章可与本站既有风格保持一致，便于直接放入 `content/posts/` 使用。
